# Graduation Project

## Description

This educational project was completed as the final assignment of the QA Engineer program at Yandex Practicum.

The project demonstrates comprehensive testing of web, API, and mobile applications, including test analysis, checklist creation, test case design, validation testing, and bug reporting.

## Objective

The objective of this project was to perform end-to-end testing of different application components, verify functionality, validate user input, identify defects, and document testing results.

## What I did

- Created UI testing checklists.
- Performed input validation testing using Equivalence Class Partitioning (ECP) and Boundary Value Analysis (BVA).
- Designed functional test cases.
- Tested REST API endpoints using Postman.
- Intercepted and modified requests using Charles Proxy.
- Performed functional and regression testing of a mobile application.
- Reported and documented defects.
- Linked test documentation with related bug reports.

## Tools

- Google Sheets
- DevTools
- Charles Proxy
- Postman
- Android Emulator

## Results

- Completed end-to-end testing of a web application, REST API, and Android mobile application.
- Applied test design techniques, including Equivalence Class Partitioning (ECP) and Boundary Value Analysis (BVA).
- Created checklists, test cases, validation tables, and bug reports.
- Used Postman, Charles Proxy, Chrome DevTools, and Android Emulator during testing.
- Prepared comprehensive testing documentation covering multiple application components.
