# API Testing

## Description

This educational project was completed as part of the QA Engineer program at Yandex Practicum.

The project demonstrates REST API testing using Postman, API checklist creation, request validation, response verification, and bug reporting.

## Objective

The objective of this project was to verify the REST API of an online store, validate request processing, check API responses, and identify functional defects.

## What I did

- Created API checklists.
- Tested REST API endpoints using Postman.
- Verified HTTP status codes and response bodies.
- Tested positive and negative scenarios.
- Validated request parameters.
- Reported and documented API defects.
- Linked API checklist results with related bug reports.

## Tools

- Postman
- Google Sheets

## Results

- Verified REST API request processing and response validation.
- Tested positive and negative API scenarios using Postman.
- Identified and documented API defects, linking them to related bug reports.
