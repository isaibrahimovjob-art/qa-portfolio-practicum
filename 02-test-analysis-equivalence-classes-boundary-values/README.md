# Test Analysis (Equivalence Classes & Boundary Values)

## Description

This educational project was completed during the QA Engineer program at Yandex Practicum.

The project includes test analysis, test design using Equivalence Class Partitioning (ECP) and Boundary Value Analysis (BVA), test case creation, bug reporting, and linking defects with supporting evidence.

## Objective

The objective of this project was to analyze system requirements, design effective test cases using test design techniques, and document defects found during testing.

## What I did

- Analyzed system requirements.
- Applied Equivalence Class Partitioning (ECP).
- Applied Boundary Value Analysis (BVA).
- Prepared test data.
- Created test cases.
- Reported and documented defects.
- Linked test cases and bug reports with supporting screenshots.

## Tools

- Google Sheets

## Results

- Analyzed system requirements and prepared test data.
- Designed test cases using Equivalence Class Partitioning (ECP) and Boundary Value Analysis (BVA).
- Documented identified defects and linked them to supporting evidence.
